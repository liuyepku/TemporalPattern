\subsection{Detailed Description}
I/\+O routines for S\+T\+L containers. 

\begin{DoxyAuthor}{Author}
(c) 1999-\/2001 Frank Hoeppner \href{mailto:frank.hoeppner@ieee.org}{\tt frank.\+hoeppner@ieee.\+org}
\end{DoxyAuthor}
This file provides functions which allow the default output {\ttfamily operator$<$$<$} with S\+T\+L container classes. In case i/o polymorphism is needed (which is not supported by {\ttfamily operator$<$$<$}), optional output functions are provided.

We provide (inline template) functions for stream input (resp.\textbackslash{} output) of the S\+T\+L data structures {\ttfamily pair}, {\ttfamily list}, {\ttfamily vector}, {\ttfamily }\mbox{[}multi\mbox{]}set, and {\ttfamily }\mbox{[}multi\mbox{]}map. All of these functions presuppose that there is a {\ttfamily operator$>$$>$} (resp. {\ttfamily operator$<$$<$}) for the underlying data types, e.\+g. for type {\ttfamily T} when {\ttfamily list$<$\+T$>$} or {\ttfamily vector$<$\+T$>$} is used. When reading a container from a stream, elements are always inserted at the end of the container, however, the end position does not have any effect if the container is ordered.
